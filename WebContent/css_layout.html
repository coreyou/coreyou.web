正常流向

在正常流向中，每個區塊層級元素都是由上往下層疊排列。這是瀏覽器處理HTML元素的預設方式，所以不需要使用CSS屬性來指定正常流向。 不過他的語法是: position: static;

DEMO

相對定位

相對定位移動元素的方式，是相對於原本正常流向的位置。
你可以使用position屬性並將值設為relative，以表明該元素要使用相對定位。
接著就可以使用top、bottom來控制距離上下邊框的距離；left、right來控制距離左右邊框的距離。
方框偏移的屬性值通常是以像素、百分比或em為單位。

p.position_relative {
 position: relative;
 top: 10px;
 left: 100px;
}

DEMO

絕對定位

當position屬性的值是absolute時，方框會被移到正常流向之外，而且不再影響頁面上的其他元素。(彷彿他們不在頁面上一樣)
position: absolute;之後可以使用偏移屬性指定與頁面邊緣的距離(top、bottom、left、right)。
大部分的瀏覽器都會在<h1>的上方預設加入邊界，如果你想移除這個邊界，可以加入margin: 0px;來解決。

h1 {
 position: absolute;
 top: 0px;
 left: 750px;
 width: 250px;
 margin: 0px;
}

DEMO

固定定位

固定定位是一種絕對定位，需要將position屬性的值設為fixed。
它會將元素放在與瀏覽器視窗相對的位置上，因此當使用者向下捲動頁面時，它就會待在完全相同的位置。

h1 {
 position: fixed;
 top: 0px;
 left: 50px;
 padding: 10px;
 width: 100%;
 margin: 0px;
}

重疊元素

z-index屬性，這個數值越高，元素就會出現在越前方，z-index有時候被稱為「堆疊上下文(stacking context)」。

h1 {
 background-color: #efefef;
 padding: 10px;
 position: fixed;
 top: 0px;
 left: 0px;
 padding: 10px;
 width: 100%;
 margin: 0px;
 z-index: -1;
}
p {
 position: relative;
 top: 70px;
 left: 70px;
 width: 750px;
 margin-top: 100px;
 z-index: 2;
}

浮動元素

float屬性，可以移走一個正常流向中的元素，將它放在容器元素的最左側或最右側，位在容器元素中的其他元素就會沿著浮動元素流動。

blockquote {
 float: right;
 width: 275px;
 font-size: 130%;
 font-style: italic;
 font-family: Georgia, Times, serif;
 margin: 70px 0px 10px 10px;
 padding: 10px;
 border-top: 1px solid #665544;
 border-bottom: 1px solid #665544;
}

使用FLOAT將元素並排

很多版面配置會將方框並排在一起，這通常是利用float屬性來達成的。
當元素浮動時，方框的高度會影像後續元素的位置，當發生位置的問題時，比較常用clear屬性來解決。

p {
 width: 230px;
 float: left;
 margin: 5px;
 padding: 5px;
 background-color: #efefef;
}

淨空浮動元素

clear屬性可以讓你表明沒有任何一個元素(同一個容器元素內)應該碰到方框的左側或右側。
此屬性可以接受以下的值:

屬性值 功能
left 方框的左側不能碰到出現在同一個容器元素內的任何其他元素。
right 方框的右側不能碰到出現在同一個容器元素內的任何其他元素。
both 方框的左側和右側都不能碰到出現在同一個容器元素內的任何其他元素。
none 元素可以碰到方框的任何一側。
下面的例子將換行過後的第一段，也就是設定class="clear"的段落，加上了clear: left;，讓它的左側不能碰到其他元素，
如此可以確保段落會做換行的動作。

p {
 width: 230px;
 float: left;
 margin: 5px;
 padding: 5px;
 background-color: #efefef;
}

浮動元素的父元素: 問題與解決方案

如果一個容器元素只包含了浮動元素，有些瀏覽器會將它視為0像素高，造成顯示異常。
這裡的例子中，div原本為1px的方框邊緣被交疊成2px的線條。
解決方法為加入兩條CSS規則: overflow: auto; 以及 width: 100%;。

div {
 margin-top: 100px;
 border: 1px solid #665544;
 overflow: auto;
 width: 100%;
}

使用浮動來製作多欄版面

許多網頁在設計中使用了多欄版面。其作法是使用一個<div>來代表每一欄，並使用下列三個CSS屬性將這些直欄並排在一起：
width(設定欄寬)、float(會將各欄並排在一起)、margin(會在欄與欄之間製造出間距)

.col1of2 {
 float: left;
 width: 550px;
 margin: 10px;

固定寬度版面

固定寬度版面不會隨著使用者縮放瀏覽器視窗而改變大小。單位大多是以像素為主。

優點 缺點
在控制元素的尺寸和定位時，像素值是很精確的。 頁面邊緣會有大空白。
比起流動版面，設計師更能掌控頁面上的項目外觀和位置。 如果使用者的螢幕解析度比設計師的螢幕解析度大，頁面可能會看起來變小，文字會較難閱讀。
你可以控制文字行的長度，不管使用者的視窗大小為何。 如果使用者增加字體大小，文字可能放不進預定的空間。
影像的尺寸會和頁面保持一定的比例。 這樣的設計通常在解析度與桌上型或筆記型電腦相似的裝置上最佳。
比起流動版面，同樣內容的頁面會占用較多的垂直空間。

body {
 width: 960px;
 margin: 0 auto;
}
#content {
 overflow: auto;
 height: 100%;
}
#nav, #feature, #footer {
 background-color: #efefef;
 padding: 10px;
 margin: 10px;
}
.column1, .column2, .column3 {
 background-color: #efefef;
 width: 300px;
 float: left;
 margin: 10px;
}
li {
 display: inline;
 padding: 5px;
}

流動版面

流動版面設計會隨著使用者縮放瀏覽器視窗而改變大小。單位大多以百分比為主。

優點 缺點
頁面會延伸填滿整個瀏覽器視窗，所以在大螢幕上，頁面兩側不會有空白。 如果你不控制頁面各區的寬度，那麼設計可能會與你當初的設計大相逕庭，某些元素周圍會出現空白，或者項目會擠在一起。
如果使用者的視窗較小，頁面可以縮小來配合，使用者不需要向側面捲動。 如果使用者的螢幕是寬的，文字行可能會變得很長以致難以閱讀。
若使用者將字體大小設得比設計師原擬的大，設計也可以容納這樣的情況。 如果使用者的螢幕是窄的，文字可能會擠在一起，導致每一行只剩幾個字。
如果固定寬度項目(例如影像)所在的方框太小不足以容納(因為使用者將視窗拉得太小)，影像可能會溢到文字上。

body {
 width: 90%;
 margin: 0 auto;
}
#content {
 overflow: auto;
}
#nav, #feature, #footer {
 margin: 1%;
}
.column1, .column2, .column3 {
 width: 31.3%;
 float: left;
 margin: 1%;
}
.column3 {
 margin-right: 0%;
}
li {
 display: inline;
 padding: 0.5em;
}
#nav, #footer {
 background-color: #efefef;
 padding: 0.5em 0;
}
#feature, .article {
 height: 10em;
 margin-bottom: 1em;
 background-color: #efefef;
}

960像素格線

格線可以在項目之間維持一致的比例和空間。有以下優點:
在不同的頁面設計之間營造延續性、在各頁面上協助使用者預期到哪裡尋找資訊、
網站加入新內容的時候可以顯得一致、幫助設計者以一致的方式合作。
DEMO中是960像素寬的12欄格線，有12個相同大小的欄，每一欄都是60像素寬，每一欄的邊界都是10像素，這樣欄與欄之間就會保持20像素的間距，頁面的左右側則是10像素的間距。

使用960.GS製作格線版面

網路上可以找到不少格線框架，這裡介紹的是960格線系統，
把它下載回電腦，解壓縮後960-Grid-System-master\code\css中有960_12_col.css可以直接套用。
我們可以在html裡面連結套用這個css檔，然後在需要的地方加入定義好css的class名稱，例如加入以下class：
class="container_12"(作為整個頁面的容器，並表明我們要使用的是12欄的格線)
class="clearfix"(以確認瀏覽器知道容器方框的高度，因為它只包含了浮動元素，這裡會使用到clear屬性)
class="grid_12"(製作了一個12欄寬的區塊，一欄60px，12欄共720px；欄間距10px，所以欄與欄之間相距20px，12欄間共有11個20px，共220px；總共940px寬)
class="grid_4"(製作了一個4欄寬的區塊，也就是400px寬)

數個樣式表

有些網頁作者會將CSS樣式分成好幾個樣式表。舉例來說，可能一個樣式表來控制版面，另一個樣式表控制字型、顏色。
有些作者會進一步使用模組化的方式來處理樣式表，也就是製作不同的樣式表來控制文字設計、版面、表單、表格，甚至網站上的每個次版塊。
有兩個方式可以在頁面上加入數個樣式表：

  1. HTML可以連到一個樣式表，然後該樣式表可以使用@import規則來匯入其他樣式表
    (和所有樣式表一樣，如果有兩條規則都套用到同一個元素，那麼後者的規則就會凌駕前者。
    使用@import那一刻起，瀏覽器才會認定規則生效，所以如果@import寫在css的最一開始，代表這個css檔中的其他規定會凌駕使用@import的規則)
  2. 在HTML中，你可以為每個樣式表使用一個獨立的<link>元素
    (和所有樣式表一樣，如果有兩條規則都套用到同一個元素，那麼後者的規則就會凌駕前者)
